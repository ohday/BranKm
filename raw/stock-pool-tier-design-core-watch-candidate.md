## 一、三层股票池体系（核心设计）

### 1. 候选池（初选池 / 基础库）

- **定位**：量化粗筛的潜力股集合
- **筛选**：
  - 基本面：营收 / 净利润 / 现金流 / ROE 硬门槛
  - 行业：景气度 + 政策 + 北向增持
  - 技术面：趋势方向 + 成交量
- **数量**：A+HK 全市场 ~7500 只 → 候选池 **100-200 只**
- **特点**：门槛低，定期淘汰财务暴雷、行业衰退、技术恶化
- **评估**：每月

### 2. 观察池（研究池 / 备选库）

- **定位**："次重仓候选"，经过深入研究
- **研究重点**：
  - 行业地位（与龙头对比）
  - 3-5 年财报纵向分析
  - 商业模式 + 现金流质量
  - 技术面验证（趋势持续、资金流向）
- **数量**：**30-50 只**
- **升级条件**：业绩反转 / 技术突破 / 政策订单催化 → 核心池

### 3. 核心池（重点池 / 实盘）

- **定位**：最终配置标的，占总仓位 **50-60%**
- **最高标准**：
  - 行业龙头或细分冠军
  - 连续 3 年盈利增长 + 低负债 + 高现金流
  - 朝阳产业（新能源 / AI / 半导体）
  - 股价位于 60 日均线上方 + 放量
- **数量**：**5-15 只**（散户）/ **20-30 只**（机构）
- **操作**：长期持有，除非基本面恶化或趋势破位

## 二、动态维护规则

| 维度 | 规则 |
|------|------|
| 仓位分配 | 核心 50-60% / 机会层（波段）30-40% / 观察 0-10% |
| 评估频率 | 每月 or 每季度 |
| 升级：候选→观察 | 通过深度研究 |
| 升级：观察→核心 | 催化剂兑现 + 技术确认 |
| 降级/剔除 | 财务暴雷 / 行业景气反转 / 趋势破位 / 止损触发 |
| 散户简化版 | 10-30 只重点，避免过度分散 |

## 三、机构 vs 散户差异

| | 机构（量化团队） | 散户（本 skill 目标用户） |
|---|---|---|
| 层级 | 基础库 → 备选库 → 核心库 → 禁买库（4 级） | 候选池 → 观察池 → 核心池（3 级） |
| 决策 | 投研会议 + 因子打分 + 风险预算 | skill + 用户手动确认 |
| 更新 | 每日因子评分 | 每周主动查看 |
| 关键点 | 避免过拟合 + 风险预算 | 明确升降级规则 + 严格执行 |

### 新手友好性设计

每条推荐必须含：
- **为什么推它**（1 句话核心逻辑）
- **亮点**（2-3 条数据支撑）
- **风险**（1-2 条反向提醒）
- **催化剂**（短期驱动事件）
- **合规提示**："本推荐仅供学习参考，非投资建议"

### 与 skill 对话的约束

- **行业排除**：用户说"不喜欢白酒" → 下次自动排除
- **偏好强化**：用户反复点"加入核心池"的股票 → 反向推断流派倾向
- **目标价与止损**：谨慎输出（新手易误读为承诺），可选开启

## 六、散户常见错误

1. **杂货铺现象**：自选股一堆但从不整理 → skill 强制"不进三层池的不推荐跟踪"
2. **核心池过大**：10 只以上无法深度跟踪 → skill 默认核心池 ≤ 5（可调）
3. **没有降级机制**：只进不出 → skill 每月自动审查降级条件
4. **混淆配置和交易**：核心池是"值得持有"，不是"今天就该买" → skill 明确区分选股和择时
