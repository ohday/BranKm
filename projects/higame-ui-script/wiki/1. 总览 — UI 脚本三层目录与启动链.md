# 总览 — UI 脚本三层目录与启动链

HiGame 是 UE5.5.4 + UnLua 架构的 DDS(分布式 DS)游戏。**UI 脚本层只在客户端实例化**,以 Lua 写 UMG 后端,通过自研 `UIManager` 单例统一管理所有 2D / 3D 界面的生命周期、栈、输入与音效。本页画出整个 UI 子系统的边界、目录结构、启动链路,后续每一页深挖其中一块。

## 三层 Lua 代码分隔与 UI 子树

HiGame 的 Lua 代码在 `Content/Script/` 下被强制分成三层,UI 几乎全部位于 **Common + Client** 层[^48]。

```mermaid
flowchart TB
    subgraph SCRIPT["Content/Script/"]
        direction TB
        COMMON["CommonScript/<br/>双端共用<br/>(Class, G, decorator, 基类, UnLua 工具)"]
        CLIENT["ClientScript/<br/>客户端专用<br/>(音频, 引导, 加载, 部分 ui/widget)"]
        SERVER["ServerScript/<br/>服务端专用<br/>(AI, 多世界, 天气) — 不含 UI"]
        UIROOT["ui/<br/>★ UI 框架与大部分 widget<br/>(uiframework, lguiframework, widget, viewmodel)"]
    end

    COMMON -. 提供 Class/Decorator .-> UIROOT
    CLIENT -. 部分 widget 也在这里 .-> UIROOT
    SERVER -. 不实例化 UI .-> X["✗"]

    style UIROOT fill:#4a9eff,color:#fff
    style SERVER fill:#888,color:#fff
    style X fill:#ff6b6b,color:#fff
```

> **路径陷阱**:业务 UI **大多数**在 `Content/Script/ui/...`(无 `ClientScript` 前缀),少数在 `Content/Script/ClientScript/ui/...`。新写 UI 推荐落 `Content/Script/ui/widget/<模块>/`。

### 运行时环境判断

```lua
UnLua.IsSSInstanceClient()                            -- 客户端
UE.UHiRuntimeEnvFunctionLibrary.IsSSInstanceGame()    -- DS 服务端
```

UI 一旦写到 `ServerScript/`,**根本不会被加载**,因为 DS 不会实例化 `UILogicSubSystem`。

`main_ui.lua` 与 `interactive_ui.lua` 已被原作者标记为废弃[^48],**新代码绝对不要拷贝它们当模板**。

## 启动链路 — 从 C++ 到 Lua UIManager

UI 子系统启动顺序非常明确:

```mermaid
sequenceDiagram
    participant W as Client World
    participant CPP as C++<br/>UUILogicSubSystem
    participant BP as BP_UILogicSubsystem<br/>(IUnLuaInterface)
    participant L as ui/ui_logic_subsystem.lua
    participant LCH as ui/ui_manager_launcher.lua
    participant DEF as UIDef
    participant VMC as ViewModelCollection
    participant MGR as UIManager (单例)

    W->>CPP: Initialize()
    CPP->>BP: 蓝图实例化
    BP->>L: GetClientModuleName 解析<br/>+ 反射方法调用
    L->>LCH: DoInitUI()
    LCH->>DEF: InitUIDef() (加载 UIInfo DataTable)
    LCH->>DEF: InitLGUIDef() (加载 LGUIInfo)
    LCH->>VMC: InitGlobalVM() (实例化所有全局 VM)
    LCH->>MGR: InitManager() (建栈/Notifier/Cursor)
    MGR-->>L: ready
    L-->>BP: PostInitializeScript
    BP-->>CPP: BlueprintImplementableEvent 完成
```

`UUILogicSubSystem` C++ 类只是骨架,**所有逻辑通过 `BlueprintImplementableEvent`**(`InitializeScript` / `PostInitializeScript` / `DeinitializeScript` / `OnWorldBeginPlayScript`)下沉到 Lua[^53]。

## UIManager — 整个 UI 系统的中央调度

`Content/Script/ui/uiframework/ui_manager.lua` 是单例,持有所有 UI 实例与子模块[^48]。

```mermaid

    subgraph MGR["UIManager (单例)"]
        direction TB
        D1["tbCreatingUI{}<br/>tbCreatedUI{}<br/>tbDestroyingUI{}<br/>(三态实例字典)"]
        D2["uiStackManager<br/>(8 层 UI 栈)"]
        D3["uiDepthMgr<br/>(ZOrder)"]
        D4["LGUIManager<br/>(3D 世界 UI<br/>延迟加载)"]
        D5["UINotifier<br/>(UI 事件总线)"]
        D6["CursorManager<br/>(光标)"]
    end

    API["核心 API:<br/>OpenUI / OpenUIByName<br/>CloseUI / CloseUIByName<br/>ReturnUI<br/>GetUIInstance<br/>GetTopUI / GetTopLGUI<br/>RegisterActionDelegate"]
    MGR --> API
```

每一块都有专门一页详解:[3. UI 栈与 Layer](3.%20UI%20栈与%20Layer.md)、[6. MVVM 数据绑定](6.%20MVVM%20数据绑定.md)、[8. UI 事件原语与 UINotifier](8.%20UI%20事件原语与%20UINotifier.md)、[9. 输入系统](9.%20输入系统.md)。
