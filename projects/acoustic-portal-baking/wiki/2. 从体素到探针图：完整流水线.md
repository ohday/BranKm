# 从体素到探针图：完整流水线

这一页把所有后续算法串成一条时间线：**从用户给定的体素网格开始，经过离线烘焙，得到可在运行时被声音系统查询的声学数据**。每一步都有独立页面详述，这里只画骨架。

## 全景图

```mermaid
flowchart TB
    subgraph OFFLINE["离线烘焙（几分钟）"]
        VOX["体素网格 + SDF<br/>（用户输入）"]
        P["① 探针布置<br/>楼层栅格<br/>→ 几百 ~ 几千 Probes"]
        VG["② 可见性图构建<br/>两两可见测试 + Euclidean cost<br/>→ Sparse AdjacencyList"]
        BAKE["③ Dijkstra 全对最短路<br/>N 次 Dijkstra 并行<br/>→ ProbePath 全对表"]
        COMPRESS["④ 压缩去重<br/>SoundPath(first, last, dist, dev)<br/>→ BakedPathData 表"]
        SAVE[("⑤ 序列化<br/>~5-20 MB 文件")]
    end

    subgraph RUNTIME["运行时（每源每帧 &lt; 100 μs）"]
        SRC["source 位置"]
        LIS["listener 位置"]
        LOC["⑥ ProbeTree BVH 查询<br/>→ 最多 8 个影响探针"]
        LOS{{"⑦ 直接 LOS ?"}}
        DIRECT["⑧a 直达路径<br/>direction, distance"]
        LUP["⑧b 查表 SoundPath<br/>×(最多 8×8 对)"]
        VAL{"⑨ 可选：校验<br/>每段重投射"}
        ASTAR["⑩ A★ 备用路径<br/>动态几何场景"]
        SHEQ["⑪ SH 方向编码<br/>+ UTD 多频段 EQ"]
        DSP["⑫ PathEffect DSP<br/>mono → Ambisonic → binaural"]
    end

    VOX --> P --> VG --> BAKE --> COMPRESS --> SAVE
    SAVE -.加载.-> LOC

    SRC --> LOC
    LIS --> LOC
    LOC --> LOS
    LOS -->|是| DIRECT
    LOS -->|否| LUP
    LUP --> VAL
    VAL -->|通过| SHEQ
    VAL -->|失败| ASTAR --> SHEQ
    DIRECT --> SHEQ
    SHEQ --> DSP

    style OFFLINE fill:#e8f5e9
    style RUNTIME fill:#fff3e0
```

## 数据量级参考

以一栋 50×50×6 m 的双层建筑为例：

```mermaid
flowchart LR
    A["体积 = 1.5万 m³<br/>体素 5cm → 120M 个体素"] --> B
    B["探针 spacing=2m<br/>→ 625 × 2 层 ≈ 1250 Probes"] --> C
    C["可见性图:<br/>pairs ≈ 78万<br/>edges ≈ 50k（稀疏 4%）"] --> D
    D["BakedPathData:<br/>refs 表 ≈ 6 MB<br/>unique paths ≈ 2 MB<br/>共 ~8 MB"] --> E
    E["烘焙时间:<br/>可见图 ~1 分钟<br/>Dijkstra ~30 秒<br/>（16 线程 CPU）"]
```

这个量级**完全在单机几分钟内可烘焙**，文件大小足以塞进关卡包。对比 Project Acoustics 的云端数小时烘焙 + 百 MB 文件，Steam Audio 派的工程可行性是巨大的优势。

## 关键约束：探针密度

整条流水线对几何的鲁棒性**完全取决于探针密度是否足够**：

```mermaid
flowchart TD
    DOOR["门宽 0.9 m"]
    PROBE["探针 spacing = 2 m"]

    DOOR -->|"门两侧各有<br/>1-2 个探针可穿过"| GOOD["✅ 可见性图捕获到这扇门"]
    DOOR -.->|"如果 spacing=3 m<br/>门两侧探针稀疏"| BAD["❌ 可能没有探针对能互见<br/>→ 这扇门被图忽略"]

    style GOOD fill:#c8e6c9
    style BAD fill:#ffcdd2
```

**经验法则**：`spacing ≤ min_aperture × 2`。用户场景里最小门宽 90 cm 对应 `spacing ≤ 1.8 m`，取 1.5 m 更保险。

## 设计选择的因果链

```mermaid
flowchart TD
    Q["为什么选探针图而非 Portal？"]
    Q --> R1["Portal 自动检测<br/>在不规则几何上不鲁棒"]
    Q --> R2["声学语义可以<br/>被两两可见性完全表达"]

    R1 --> D1["→ 探针密度是唯一调节量"]
    R2 --> D2["→ 无需任何几何后处理"]

    D1 --> IMPL["实现"]
    D2 --> IMPL

    IMPL --> F1["对复杂建筑形态不敏感<br/>（中庭、夹层、楼梯井都可）"]
    IMPL --> F2["对家具级遮挡物鲁棒<br/>（看得见就连边）"]
    IMPL --> F3["动态几何可热更新<br/>（A★ 重搜）"]
```

详细对比见 [12. 方法对比与原型建议](12.%20方法对比与原型建议.md)。
