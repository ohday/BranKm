# GameplayCue 表现层

GameplayCue(简称 GC)是 GAS 的"客户端表现"分发机制 — 服务端只发"应该播什么",每个客户端各自决定播放细节。HiGame 在 GAS 标准 Cue 之上做了**两类增强**:Lua 化的 Cue 子类(子蒙太奇/相机抖动/Sequence)、`UHiGameplayCueManager` 关闭启动时的 AsyncLoad(改为按需加载)。本页讲清 Cue 类层次、Lua 端 GC 模板、与蒙太奇/Niagara/Wwise 的整合点[^c01][^c05]。

## GameplayCue 基本概念

```mermaid
flowchart TB
    GE["UGameplayEffect.GameplayCues"]
    SPEC["FGameplayEffectSpec.GetGrantedTags<br/>+ FGameplayEffectCue"]
    APPLY["ASC.ApplyGameplayEffectSpecToSelf"]
    EVENT{"事件类型"}
    NS["AGameplayCueNotify_Static<br/>(蓝图层无状态,EventType 一次)"]
    NA["AGameplayCueNotify_Actor<br/>(可生成 Actor / WhileActive)"]

    GE --> SPEC --> APPLY
    APPLY --> EVENT
    EVENT -->|OnExecute<br/>(Burst,瞬发)| NS
    EVENT -->|OnExecute / Add / Remove<br/>WhileActive| NA

    NS --> Effect1["播 Niagara / Sound / SpawnDecal"]
    NA --> Actor1["挂 Niagara/Sound 子组件,持续到 Remove"]

    style EVENT fill:#4a9eff,color:#fff
```

| 类型 | 触发 | 用途 |
|------|------|------|
| `Notify_Static` | OnExecute(瞬发)/ OnExecute_Implementation | 命中爆炸/打击特效/伤害飘字 |
| `Notify_Actor` | OnAdd / OnRemove / WhileActive | Buff 持续特效(中毒身上的烟、护盾光圈) |
| 项目自研 | 子蒙太奇 / 相机抖动 / Sequence | 见下文 |

## HiGameplayCueManager — 关闭启动 AsyncLoad

```cpp
// Public/HiAbilities/HiGameplayCueManager.h
UCLASS(config = Game)
class HIGAME_API UHiGameplayCueManager : public UGameplayCueManager
{
public:
    virtual bool ShouldAsyncLoadObjectLibrariesAtStart() const override { return false; }
    virtual TArray<FString> GetValidGameplayCuePaths() override
    {
        TArray<FString> result;
        return result;
    }
};
```

> **设计理由**:GAS 默认会扫描整个项目目录下所有 `Notify_Static`/`Notify_Actor` 蓝图并 AsyncLoad,**HiGame 项目几千个 Cue 全部启动期加载会让登陆耗时增加几秒**。改为返回空路径 + 关闭启动 Async,所有 Cue 改为**首次 Apply 时按需 LoadObject**。
> 副作用:第一次触发某 Cue 会有几帧延迟,但游戏内大部分 Cue 在动画/技能预加载阶段就被引用并加载完。

## Lua 化 Cue 实现

`Content/Script/skill/GameplayCues/` 下有项目自研的 Lua Cue 实现,**通过蓝图 Notify_Static + IUnLuaInterface 绑定到 Lua 模块**:

```
Content/Script/skill/GameplayCues/
├── GC_SetMontagePlayRate.lua          ← 命中减速(命中瞬间慢一帧两帧)
├── gc_sub_camerashake.lua              ← 命中相机抖动
└── gc_sub_sequence.lua                  ← 命中播过场镜头(子 Sequence)
```

### GC_SetMontagePlayRate(Lua 控制蒙太奇时间)

```lua
-- skill/GameplayCues/GC_SetMontagePlayRate.lua
-- 用途:命中瞬间把攻击方动画的 PlayRate 临时降为 0.1,持续 X 帧后恢复
-- (打击感"凝滞"经典效果)

local GC_SetMontagePlayRate = Class()

function GC_SetMontagePlayRate:OnExecute_Implementation(MyTarget, Parameters)
    local Source = Parameters.Instigator
    if not Source or not Source:IsValid() then return end

    -- 1. 拿到当前播放的蒙太奇
    local AnimInstance = Source.Mesh:GetAnimInstance()
    if not AnimInstance then return end
    local Montage = AnimInstance:GetCurrentActiveMontage()
    if not Montage then return end

    -- 2. 从 Parameters 拿配置 (RawMagnitude=多少帧 / NormalizedMagnitude=慢速倍率)
    local Frames = Parameters.RawMagnitude or 4
    local SlowRate = Parameters.NormalizedMagnitude or 0.1

    -- 3. 立即改 PlayRate
    AnimInstance:Montage_SetPlayRate(Montage, SlowRate)

    -- 4. 计时恢复
    UE.UKismetSystemLibrary.K2_SetTimer(Source, function()
        AnimInstance:Montage_SetPlayRate(Montage, 1.0)
    end, Frames / 60.0, false)
end

return GC_SetMontagePlayRate
```

### gc_sub_camerashake(Lua 控制相机抖动)

```lua
-- skill/GameplayCues/gc_sub_camerashake.lua
-- 用途:从 Parameters 中读取 CameraShake 类与强度,对所有玩家相机施加抖动

local GC_SubCameraShake = Class()

function GC_SubCameraShake:OnExecute_Implementation(MyTarget, Parameters)
    local CameraShakeClass = Parameters.SourceObject     -- 类引用通过 Parameters.SourceObject 传
    if not CameraShakeClass then return end

    local World = MyTarget:GetWorld()
    local PlayerController = UE.UGameplayStatics.GetPlayerController(World, 0)
    if PlayerController then
        PlayerController:ClientStartCameraShake(CameraShakeClass, Parameters.NormalizedMagnitude)
    end
end

return GC_SubCameraShake
```

### gc_sub_sequence(Lua 触发子 Sequence)

```lua
-- skill/GameplayCues/gc_sub_sequence.lua
-- 用途:命中目标后播一段 LevelSequence(必杀镜头特写)

local GC_SubSequence = Class()

function GC_SubSequence:OnExecute_Implementation(MyTarget, Parameters)
    local SequenceAsset = Parameters.SourceObject
    if not SequenceAsset then return end

    local Bindings = ...    -- 把攻击方/受击方绑到 Sequence 的 Possessable
    UE.ULevelSequencePlayer.CreateLevelSequencePlayer(MyTarget, SequenceAsset, ...,
        function(Player)
            Player:Play()
        end)
end

return GC_SubSequence
```

> 这三个 Lua Cue 都是**通用工具型** — 蓝图只配 `Tag + 携带的 SourceObject(Sequence/CameraShake)/Magnitude`,Lua 决定具体怎么播。

## FGameplayCueParameters 字段速查

```cpp
struct FGameplayCueParameters
{
    UPROPERTY() float NormalizedMagnitude;      // 0~1 (用于淡入淡出)
    UPROPERTY() float RawMagnitude;             // 原始数值(伤害/数量)
    UPROPERTY() FGameplayEffectContextHandle EffectContext;
    UPROPERTY() FGameplayTag MatchedTagName;
    UPROPERTY() FGameplayTag OriginalTag;
    UPROPERTY() FGameplayTagContainer AggregatedSourceTags;
    UPROPERTY() FGameplayTagContainer AggregatedTargetTags;
    UPROPERTY() FVector Location;               // 命中点
    UPROPERTY() FVector Normal;                 // 命中法线
    UPROPERTY() FGameplayEffectContextHandle EffectContext;
    UPROPERTY() FName GameplayCueTag;
    UPROPERTY() TWeakObjectPtr<AActor> Instigator;
    UPROPERTY() TWeakObjectPtr<AActor> EffectCauser;
    UPROPERTY() TWeakObjectPtr<const UObject> SourceObject;     // ★ 万能传引用
    UPROPERTY() FHitResult HitResult;                            // ★ 命中点(穿透/法线)
    UPROPERTY() bool bReplicateLocationWhenUsingMinimalRepProxy;
    UPROPERTY() TArray<TObjectPtr<AActor>> TargetAttachComponent;
    -- ...
};
```

> `SourceObject` 是项目用得最多的字段 — 因为 GameplayCueParameters **不能携带任意类型**,通常通过 SourceObject 传 Niagara System / Sequence Asset / CameraShake Class 等。

## Cue 触发的 4 种方式

```cpp
// ASC 提供的 Cue 触发 API

// 1. ExecuteGameplayCue (RPC, 客户端 OnExecute)
void ExecuteGameplayCue(FGameplayTag GameplayCueTag,
                         const FGameplayCueParameters& Parameters);

// 2. AddGameplayCue (RPC, 客户端 OnAdd, 持续)
void AddGameplayCue(FGameplayTag GameplayCueTag,
                     const FGameplayCueParameters& Parameters);

// 3. RemoveGameplayCue
void RemoveGameplayCue(FGameplayTag GameplayCueTag);

// 4. RemoveAllGameplayCues
void RemoveAllGameplayCues();
```

> Lua 触发:`UHiGASLibrary.ExecuteGameplayCue(Actor, Tag, Parameters)` 等便捷封装。

## GE 内联 Cue(配置式)

GE 蓝图层 `GameplayEffectComponents` / `Display` 可以直接挂 `FGameplayEffectCue`:

```
GE_Damage_Slash01_C:
  GameplayEffectComponents:
    UGameplayCueNotifyComponent (内置的 Cue 组件):
      Cues:
        - GameplayCueTag: GameplayCue.Hit.Slash
          MagnitudeAttribute: Damage
          MinLevel: 0  MaxLevel: 100
        - GameplayCueTag: GameplayCue.SlowMo.Frame4
        - GameplayCueTag: GameplayCue.CameraShake.SlashHit
```

GE 应用(命中)时这些 Cue 自动 Execute。`MagnitudeAttribute` 让 Cue 拿到该属性的实际值(用于"伤害越高越大特效")。

## Buff 内联 Cue(DataTable)

```cpp
// FHiBuffRow 的 DynamicGameplayCues
UPROPERTY(EditAnywhere, BlueprintReadWrite)
TArray<FGameplayEffectCue> DynamicGameplayCues;
```

`MakeBuffEffectSpec` 时这些 Cue 被加到 Spec,GE 应用与移除时自动:
- OnAdd → 持续效果(中毒紫色烟雾)
- OnRemove → 结束效果
- WhileActive → 周期触发

## 性能注意

### 1. 不要 Spawn-and-Destroy 频繁

Notify_Actor 默认每次 OnExecute 都 Spawn 一个 Actor,对每帧上万次的命中(怪物挥砍 + 玩家被打)会拖慢。**项目惯例:用 Notify_Static + 内部 ObjectPool**。

### 2. AsyncLoad 已关闭

`UHiGameplayCueManager` 启动时不预加载,**首次 Apply 时 LoadObject 阻塞主线程几帧**。
解法:在 `UHiGameplayAbility.LoadAssets/LoadClasses` 里把技能用到的 Cue 类列出来,GA 第一次 Activate 时异步加载,后续命中时已就绪。

```cpp
// Public/HiAbilities/HiGameplayAbility.h
UPROPERTY(EditDefaultsOnly, Category="Asset Settings")
TArray<TObjectPtr<UObject>> LoadAssets;     // 预加载资源
UPROPERTY(EditDefaultsOnly, Category="Asset Settings")
TArray<TSubclassOf<UObject>> LoadClasses;
```

### 3. WhileActive 用法限制

`Notify_Actor.WhileActive` 客户端每帧 Tick,**多个 Buff 同时挂可能性能炸**。中毒/燃烧这类长期 Buff 改用 `OnAdd`(Spawn 子 Niagara 持续) + `OnRemove`(Destroy)。

## 与蒙太奇/Wwise/Niagara 整合

```mermaid
flowchart LR
    subgraph CueChain["GameplayCue 触发链"]
        TAG[GameplayCue.Hit.Slash]
        TAG --> CUE[Notify_Static / Lua]
    end

    subgraph Effect["最终表现"]
        CUE --> NIA[Niagara<br/>(粒子, 命中爆炸)]
        CUE --> WWISE[Wwise AKEvent<br/>(打击声 + 武器声)]
        CUE --> CAM[相机抖动<br/>UCameraShakeBase]
        CUE --> MONT[蒙太奇 PlayRate<br/>(打击凝滞)]
        CUE --> SEQ[LevelSequence<br/>(必杀镜头特写)]
    end

    style TAG fill:#4a9eff,color:#fff
```

各模块责任:
- **Niagara** — Niagara System 通过 `UNiagaraComponent.SpawnSystem*` API 在命中点 Spawn,FX/VFX 团队配资源
- **Wwise** — `AkAudio` 模块的 `UAkComponent` PostEvent;Cue 内部根据 Tag/SkillType 分流不同 Event
- **CameraShake** — `UCameraShakeBase` 子类,用 `PlayerController.ClientStartCameraShake`
- **蒙太奇 PlayRate** — Lua 实现的 `GC_SetMontagePlayRate.lua`(见上文)
- **LevelSequence** — Lua 实现的 `gc_sub_sequence.lua`(见上文)

## 一页速查

| 任务 | 方法 |
|------|------|
| 命中播一次特效 | GE 内挂 Cue(Notify_Static),`OnExecute` 走 Niagara 一发 |
| Buff 期间持续特效 | DataTable `DynamicGameplayCues`(Notify_Actor),`OnAdd/Remove/WhileActive` |
| 命中减速 | `GameplayCue.SlowMo.Frame4` 走 `GC_SetMontagePlayRate` |
| 相机抖动 | `gc_sub_camerashake` + `Parameters.SourceObject = CameraShakeClass` |
| 必杀过场 | `gc_sub_sequence` + `Parameters.SourceObject = LevelSequence` |
| 防止首次播放卡顿 | GA `LoadAssets / LoadClasses` 预加载 Cue 资源 |
| Cue 不触发 | 检查 GE 是否真应用、Tag 是否在 GameplayCue 父级、Notify 路径是否对 |
| 性能优化 | 用 Notify_Static + 项目对象池;避免 WhileActive 给所有玩家挂 |
