# 配色与字体系统

> **残酷真相**：截至 2026.5，**没有任何 AI 网页生成工具真的应用色彩理论 + 字体配对算法**。它们要么是"调色板选择题"，要么是"模型直觉"，要么是"启发式规则"。这一页解释为什么、各家走的什么路、以及如何**绕开**这个限制。

## 三种"伪自动化"路线

```mermaid
flowchart TB
    Approach["AI 工具的配色机制"]
    Approach --> R1["①模板/调色板选择"]
    R1 --> R1a["v0 / shadcn:12 板调色板"]
    R1 --> R1b["Stitch:Material 主题 token"]
    R1 --> R1c["Lovable 2.0:Visual Edits 手选"]

    Approach --> R2["②LLM 直觉(无算法)"]
    R2 --> R2a["Bolt.new (Claude 直接生成)"]
    R2 --> R2b["Trae SOLO (中文 prompt 选色)"]
    R2 --> R2c["DeepSite V2 (DeepSeek 直觉)"]

    Approach --> R3["③参考图复刻(多模态)"]
    R3 --> R3a["v0:Figma + screenshot 导入"]
    R3 --> R3b["Lovable:多模态文件分析"]
    R3 --> R3c["Readdy:点选改色 + 上传参考"]
    R3 --> R3d["Stitch:image-to-UI 模式"]

    Approach --> R4["④真色彩理论"]
    R4 --> R4a["❌ 没有任何工具做这个"]
```

## v0 / shadcn 生态：12 板选择题

shadcn 的设计哲学是"give you defaults, not opinions"——给你 12 个调色板，自己挑[^61][^63]：

```mermaid
flowchart LR
    Pal["shadcn 调色板"]
    Pal --> Neutral["中性色"]
    Neutral --> N1["slate"]
    Neutral --> N2["zinc"]
    Neutral --> N3["stone"]
    Neutral --> N4["gray"]
    Neutral --> N5["neutral"]

    Pal --> Color["彩色"]
    Color --> C1["red"]
    Color --> C2["rose"]
    Color --> C3["orange"]
    Color --> C4["green"]
    Color --> C5["blue"]
    Color --> C6["yellow"]
    Color --> C7["violet"]

    Pal --> Mode["模式"]
    Mode --> M1["Default(flat)"]
    Mode --> M2["New York(精致)"]
```

**结果是同质化**：所有 v0 + shadcn 站点远看都长得差不多。优点是稳，缺点是没"个性"。

> 想破局？用 **TweakCN**（AI 驱动的 shadcn 主题编辑器）：自然语言改色 / 圆角 / 字号，导出 CSS 变量到 shadcn 项目[^63]。

## Lovable 2.0 Visual Edits：半自动启发式

Lovable 是这批里**唯一在字体配对上做了启发式规则**的工具[^63]：

```mermaid
flowchart TD
    L["Lovable 2.0 设计系统"]
    L --> Color["配色"]
    Color --> CR1["Visual Edits 模式手选"]
    Color --> CR2["推荐用 mycolor.space 定品牌色"]
    Color --> CR3["示例: pink #f7deda + 互补 #7fd1ae + #ff7b8e"]

    L --> Font["字体"]
    Font --> FR1["支持 Google Fonts + web-safe"]
    Font --> FR2["可 prompt 指定: Playfair Display 标题 / Verdana 正文"]
    Font --> FR3["半自动规则:每项目 ≤3 字体"]
    Font --> FR4["半自动规则:serif headline + sans body 启发式对比"]

    L --> Token["设计 token"]
    Token --> T1["8px 间距网格"]
    Token --> T2["全局字体大小/spacing token"]

    L --> Trend["跟 2026 趋势"]
    Trend --> TR1["Pantone 'Cloud Dancer' 米白"]
    Trend --> TR2["citrus / fuchsia 撞色"]
    Trend --> TR3["大字号 hero + 简洁 body"]
```

## Readdy：点选 + 参考图

Readdy 的配色不来自算法，但**编辑体验最接近设计师工具**[^62]：

- **Point-to-point editing**：点任意元素改色（"红渐变 → 靛青"）
- 选区改配色 / 替换图文 / 加纹样
- 上传参考图复刻：实测复刻抖音国风活动页，按钮和纹理接近原稿
- 暗色切换瞬时换板
- 每次编辑保存为新版本可对比回滚

但 review 中**未发现真正的"参考图色彩提取算法"** —— 它是 LLM 看图后的"再生"而非"提取"。

## 复刻参考图配色：哪个工具能做？

```mermaid
flowchart LR
    Ref["你有一张参考图想复刻配色"]

    Ref --> Yes["✅ 能做"]
    Yes --> Y1["Lovable<br/>多模态文件分析"]
    Yes --> Y2["v0<br/>Figma + image-to-code"]
    Yes --> Y3["Readdy<br/>实测按钮纹理接近原稿"]
    Yes --> Y4["Trae SOLO<br/>截图标注 + Sketch 导入"]
    Yes --> Y5["Stitch<br/>image-to-UI 模式"]
    Yes --> Y6["Claude Artifacts<br/>多模态但需 prompt 明确"]

    Ref --> No["❌ 不行"]
    No --> N1["Framer<br/>主要靠 prompt 描述"]
    No --> N2["Bolt.new<br/>无 image-to-code"]
```

> **实操建议**：要"复刻配色"时，最稳的路径不是依赖工具的"自动复刻"，而是：
> 1. 用 Coolors / Adobe Color 提取参考图主色板
> 2. 把 hex 值喂给工具的 prompt（"primary: #f7deda, accent: #ff7b8e"）
> 3. 让工具按你给定的 token 生成

## 2026 配色趋势锚点（参考用）

工具不会替你做色彩理论，但你可以让 prompt 跟趋势：

| 来源 | 趋势 |
|------|------|
| Pantone 2026 | Cloud Dancer 米白主基 + citrus / fuchsia 撞色[^63] |
| Framer 默认 | 暗色 dominance + Liquid Glass 雾化质感[^62] |
| 国风营销页 | 水墨色调 + 书法字体 + 留白节奏[^62] |
| 商务 SaaS | 柔和 gradient + neutrals + accent color[^61] |

## 实战 prompt 模板（绕开"工具不懂色彩"）

```
1. 给定主色 + 辅色 + 强调色 hex 值
2. 给定字体名 (Google Fonts 名)
3. 给定大字号 hero 期望大小 (e.g., 72px)
4. 给定 spacing token (8px 网格)
5. 给定参考站点 URL 或截图
```

把"色彩理论"自己做完再喂给 AI，比期望它"自动配色"靠谱 10 倍。
