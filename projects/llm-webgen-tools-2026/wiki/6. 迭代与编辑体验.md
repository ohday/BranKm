# 迭代与编辑体验

视觉漂亮的初稿一文不值——能不能"改成你要的样子"才是工具的真功夫。本页拆解每家工具的迭代回路：是 prompt-only、还是 element-level、还是 Figma 级 WYSIWYG？

## 三种迭代范式

```mermaid
flowchart TB
    Iter["AI 网页生成的迭代范式"]

    Iter --> P1["①Prompt-only<br/>(对话即编辑)"]
    P1 --> P1a["v0:NL prompt 改 Tailwind 类"]
    P1 --> P1b["Bolt.new:整段 prompt 重写"]
    P1 --> P1c["DeepSite V2:Diff Patching 增量(~3s)"]
    P1 --> P1d["Trae SOLO:中文指令"]
    P1 --> P1e["Claude Artifacts:多轮对话"]

    Iter --> P2["②Element-level<br/>(点元素改)"]
    P2 --> P2a["Lovable:Figma-like canvas + 实时同步代码"]
    P2 --> P2b["Readdy:Point-to-point + 版本历史"]
    P2 --> P2c["Subframe:拖拽 + AI prompt 双轨"]

    Iter --> P3["③Multimodal<br/>(图/文/语音)"]
    P3 --> P3a["v0:Figma + 截图导入"]
    P3 --> P3b["Lovable:多模态文件分析"]
    P3 --> P3c["Stitch:Voice Canvas 语音改图"]
    P3 --> P3d["Trae:截图标注 + PDF 导入"]
    P3 --> P3e["Readdy:参考图复刻"]
```

## Lovable：迭代体验最丰富

Lovable 在 2025 引入 Visual Edits 后，迭代体验已经追上设计工具[^61][^63]：

```mermaid
flowchart TD
    L["Lovable 编辑回路"]
    L --> Mode1["Build 模式"]
    Mode1 --> M1["生成 / 修改 (0.5 / 2+ credits)"]

    L --> Mode2["Chat 模式"]
    Mode2 --> M2["计划 / debug / 问答 (1 credit/msg)"]

    L --> Mode3["Plan 模式 ⭐"]
    Mode3 --> M3["AI 先列 schema / 架构方案<br/>再让你确认"]
    Mode3 --> M3b["避免上来就乱改"]

    L --> Mode4["Visual Edits 模式 ⭐"]
    Mode4 --> M4a["Figma-like canvas"]
    Mode4 --> M4b["拖拽 + 调 layer 顺序"]
    Mode4 --> M4c["style 面板"]
    Mode4 --> M4d["实时同步到代码"]

    L --> Multi["多模态"]
    Multi --> Mu1["拖图进 chat"]
    Multi --> Mu2["GitHub 仓库图片引用"]
    Multi --> Mu3["截图/PDF/设计稿分析"]
```

**Plan Mode** 是 Lovable 的差异化核心：在做大改前先让 AI 输出"我打算改 schema 这几张表 / 架构这样改"，避免"对话 100 次还没出 MVP"的窘境（用户实测：150 条消息没出 MVP 是常见情况[^61]）。

## Readdy：版本历史 + 点选

Readdy 的迭代设计学的是 Figma + Notion[^62]：

```mermaid
flowchart LR
    R["Readdy 编辑流"]
    R --> Click["点元素"]
    Click --> CR["改图 / 字 / 色"]
    CR --> Save["每次保存为新版本"]
    Save --> Compare["可侧栏对比"]
    Compare --> Restore["可回滚"]

    R --> Continue["继续生成功能"]
    Continue --> Sub["生成子页面"]
    Sub --> Style["保持风格一致"]

    R --> Ref["上传参考图"]
    Ref --> Replicate["复刻按钮 / 纹理 / 配色"]
```

## v0：开发者风格的"prompt 即编辑"

v0 没有 WYSIWYG 编辑器，但精度高[^61][^63]：

- "选中 navbar，改成黑色"——AI 直接改对应 Tailwind 类
- 多轮对话有上下文记忆，但**跨文件依赖记不住**
- 最近支持 Figma 直接导入，导入后改了 v0 也跟着改
- 适合开发者：精准、可追溯、与 git 友好

## Stitch：语音改图（Voice Canvas）

Stitch 是这批里**唯一支持语音迭代**的工具[^63]：

```mermaid
flowchart LR
    S["Stitch 多模态输入"]
    S --> Text["文本: '深色健身 app + tab + 进度图'"]
    S --> Voice["语音: '把字号调大'"]
    S --> Sketch["手绘草图 → UI"]
    S --> Image["截图 → UI"]

    Voice --> RT["实时调整"]
    RT --> Out["输出更新"]
```

但语音模式（和 sketch-to-UI）**不支持 Figma 导出**[^63]——只有 Standard Mode 才能完整导出。

## Trae SOLO 的中文友好迭代

Trae 的差异化在中文 prompt 的精度[^62]：

```mermaid
flowchart TD
    T["Trae SOLO 迭代"]
    T --> CN["中文 prompt"]
    CN --> CN1["'粉色渐变背景 + 飘落爱心动画'"]
    CN --> CN2["中文注释 98.7% 准确率"]

    T --> Multi["多模态输入"]
    Multi --> M1["Figma 设计稿"]
    Multi --> M2["Sketch 设计稿"]
    Multi --> M3["PDF / Word 文件"]
    Multi --> M4["截图标注 (看图写代码)"]

    T --> Realtime["实时预览 + NL 调整"]
    Realtime --> R1["'背景改星空,爱心换流星'"]

    T --> Limit["2026.4 独立端不能改代码<br/>需回退 IDE 版"]
```

## Subframe：设计 / 代码同源画布

Subframe 是为"设计师 + 开发"团队做的[^62]：

- 拖拽组件 + AI prompt 双轨
- 设计师画 → 自动生成 React/Tailwind 代码
- pixel-perfect 输出
- 自定义组件可同步到代码
- $29/编辑器/月，免费档可探索

## "Diff Patching" 加速迭代（DeepSite V2 创新）

```mermaid
flowchart LR
    Old["传统 LLM 改代码"]
    Old --> O1["整页重写"]
    Old --> O2["响应慢 (10s+)"]
    Old --> O3["可能改坏未涉及部分"]

    New["DeepSite V2 Diff Patching"]
    New --> N1["仅算 diff"]
    New --> N2["~3s 响应"]
    New --> N3["保留未改部分"]
```

这个机制现在被多家工具借鉴。

## Claude Artifacts 的迭代局限

```mermaid
flowchart TD
    CA["Claude Artifacts 迭代"]
    CA --> Way1["✅ 多轮对话"]
    CA --> Way2["✅ 多模态 (上传图)"]
    CA --> Way3["✅ 单元素改 (描述要改的)"]
    CA --> Limit["❌ 不能"]
    Limit --> L1["跳到代码某行直接改"]
    Limit --> L2["拖拽 / 视觉编辑"]
    Limit --> L3["跨 Artifact 共享状态"]
    Limit --> L4["debug 时连本地工具链"]
```

## 实战：迭代效率对比

| 任务 | v0 | Lovable | Bolt | Readdy |
|------|----|---------|------|--------|
| 改一个按钮颜色 | 1 prompt 30s | 点选改 5s | 1 prompt 60s | 点选改 5s |
| 加一个 hero section | 1 prompt 1min | 1 prompt 1min | 1 prompt 2min | 点 + Continue 30s |
| 复刻参考图 hero | Figma 导入然后改 | 拖图进 chat | 不支持 | 上传参考图 30s |
| 改全站配色 | 改 globals.css | 改 design tokens | 全文修改 prompt | 全局换主题 |
| 局部布局调整 | prompt 风险高 | Visual Edits 拖 | prompt 风险高 | 点选改 |
