# 翻车场景清单

每个工具都会翻车，关键是**翻在你不打算用的地方**。本页按工具列出已被实测/社区验证的失败模式，帮你提前避开雷区。

## 翻车类型总览

```mermaid
flowchart TB
    Fail["AI 网页生成翻车谱系"]
    Fail --> F1["💸 成本翻车<br/>(token 爆炸)"]
    Fail --> F2["🎨 视觉翻车<br/>(动画掉帧 / 配色不和谐)"]
    Fail --> F3["🏗️ 架构翻车<br/>(上下文丢失 / 跨文件断)"]
    Fail --> F4["🚪 平台翻车<br/>(锁定 / 导出残缺)"]
    Fail --> F5["🔌 集成翻车<br/>(后端 / API / 部署)"]
    Fail --> F6["🌍 访问翻车<br/>(国内 / 网络)"]
```

## 跨工具的"AI 写动画"通病

```mermaid
flowchart TB
    Common["AI 生成动画代码通病"]

    Common --> CSS_["CSS @keyframes"]
    CSS_ --> CSS1["复杂 DOM 不加 will-change → 掉帧"]
    CSS_ --> CSS2["动画 top/left 而非 transform → layout thrashing"]
    CSS_ --> CSS3["缺 requestAnimationFrame 生命周期 → 内存泄漏"]

    Common --> GSAP_["GSAP"]
    GSAP_ --> G1["Timeline chain 锚点错乱"]
    GSAP_ --> G2["无脑加 will-change 内存膨胀"]
    GSAP_ --> G3["forced reflow"]

    Common --> FM_["Framer Motion"]
    FM_ --> F1["FLIP layout overhead"]
    FM_ --> F2["gesture 跑主线程"]
    FM_ --> F3["AnimatePresence 边界 fragile"]
    FM_ --> F4["spring 难调"]

    Common --> ThreeJS_["Three.js"]
    ThreeJS_ --> T1["浏览器无 GPU 硬件加速控制"]
    ThreeJS_ --> T2["高面数模型掉帧"]
    ThreeJS_ --> T3["缺 InstancedMesh / 纹理压缩"]
```

## 常见翻车的 workaround 速查

| 翻车 | workaround |
|------|-----------|
| Lovable 物理动画 | `transform: translateZ(0)` 强制 GPU 合成；预 bake 物理 |
| Bolt 大项目 OOM | 拆模块 + Lottie 用 `renderer: 'canvas'` |
| v0 frame drop | 给关键元素加 `will-change: transform` |
| Bolt token 爆 | 切小项目 + 频繁开新 chat |
| Lovable 没出 MVP | 用 Plan Mode 先列方案再执行 |
| Framer 想要代码 | 试 Unframer (React 19 + 部分动画限制) |
| Readdy 国风弱 | 自备字体 + 上传参考图复刻 |
| Trae 复杂项目 | 切回 IDE 版 + 人工补后端 |
| Artifacts 多文件 | 拆成多 Artifact + 复制粘贴 |
| 国内访问 | 见 [9. 国内可访问性专题.md](9.%20国内可访问性专题.md) |
