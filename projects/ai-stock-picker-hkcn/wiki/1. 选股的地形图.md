# 选股的地形图

选股（Stock Selection）和择时（Market Timing）是两个独立的量化问题。本项目的 skill **只解决"买什么"**，不解决"何时买"。在深入任何具体方法之前，这一页先把"选股应该关注哪些方面"的完整空间摊开来给你看——这样你在与 skill 对话时，心里有地图。

## 选股的五条分岔

```mermaid
flowchart TD
    ROOT["选股 Stock Selection"]

    ROOT --> A1["① 看什么<br/>关注哪些维度"]
    ROOT --> A2["② 用什么方法<br/>流派选择"]
    ROOT --> A3["③ 持多久<br/>周期决定权重"]
    ROOT --> A4["④ 哪个市场<br/>A 股 vs 港股"]
    ROOT --> A5["⑤ 怎么输出<br/>候选池格式"]

    A1 --> D1["基本面 · 技术面 · 量价 · 资金流 · 风险"]
    A2 --> D2["价值 / 成长 / 质量 / 红利 / 动量"]
    A3 --> D3["长线（半年+）· 中线（1-6 月）· 波段（1-6 周）"]
    A4 --> D4["A 股特色 vs 港股特色<br/>两套市场规则"]
    A5 --> D5["候选池 · 观察池 · 核心池"]
```

每一条分岔都会深刻改变 skill 的输出。**没有统一答案**——答案由你的「画像」决定。

## 自上而下 vs 自下而上

```mermaid
flowchart LR
    subgraph TD["Top-down（自上而下，本项目主路径）"]
        M1["宏观周期<br/>美林时钟"] --> I1["行业轮动<br/>六维景气度"] --> S1["行业内个股<br/>12 维打分"]
    end

    subgraph BU["Bottom-up（自下而上，补充）"]
        G1["全市场扫描<br/>多因子 Alpha"] --> P1["个股研究<br/>护城河评估"] --> PT1["组合构建"]
    end

    TD -.适合.-> CASE1["A+HK 市场<br/>行业轮动明显<br/>政策驱动强"]
    BU -.适合.-> CASE2["长期持股<br/>Buffett 式<br/>忽略宏观"]
```

**为什么本 skill 以 Top-down 为主**？因为你的典型自然语言查询就是 Top-down：

- "**当前最值得关注的行业**是什么" → 先判行业
- "**消费行业里**给我推 5 只长线股" → 先锁定行业再选股

这是 Top-down 的原生入口。Bottom-up 的"全市场打分排序"对新手不友好，且在 A 股政策市中容易跑偏[^34][^36]。

## 五大流派的"关注维度"各不相同

这是最容易被新手忽视的事实：**价值派和动量派看的根本不是同一张表**。

```mermaid
flowchart TB
    subgraph V["价值派"]
        V1["PE / PB<br/>历史分位"] --> V2["安全边际"]
    end

    subgraph G["成长派"]
        G1["营收/净利增速<br/>PEG"] --> G2["未来 3 年空间"]
    end

    subgraph Q["质量派"]
        Q1["ROE / ROIC<br/>现金流"] --> Q2["护城河"]
    end

    subgraph D["红利派"]
        D1["股息率<br/>分红连续性"] --> D2["现金流造假否"]
    end

    subgraph MOM["动量派"]
        MOM1["相对强度 RPS<br/>突破新高"] --> MOM2["资金流入"]
    end

    V -.不看.-> MOM1
    MOM -.不看.-> V1
```

一个不加选派的「多因子打分」，结果是价值因子和动量因子互相抵消，**没得分最高，没得分最低，全在中间**——这是新手 skill 最容易踩的坑。因此 skill 必须先通过 Socratic 问卷推断你的流派倾向，再激活对应维度的权重。

## 周期决定一切

```mermaid
flowchart LR
    subgraph LONG["长线（半年+）"]
        L1["基本面 70%"] & L2["技术面 10%"] & L3["资金面 5%"] & L4["其他 15%"]
    end

    subgraph MID["中线（1-6 月）"]
        M1["基本面 45%"] & M2["技术面 25%"] & M3["资金面 20%"] & M4["其他 10%"]
    end

    subgraph SWING["波段（1-6 周）"]
        S1["基本面 15%"] & S2["技术面 30%"] & S3["资金面 30%"] & S4["催化剂 25%"]
    end

    LONG --> MID --> SWING
```

**同一只股票，在不同周期下应不应该买是两个完全不同的问题**。skill 启动时会问你选择哪个周期，然后激活对应的权重矩阵（详见 [5. 个股 12 维度体系 + 周期权重](5.%20个股%2012%20维度体系%20%2B%20周期权重.md)）。

## A 股 vs 港股：不是"同一个市场说不同语言"

| 维度 | A 股 | 港股 |
|------|------|------|
| 交易制度 | T+1 | T+0 |
| 涨跌幅 | ±10% / ±20% / ±30% | 无限制 |
| 个人融券 | 50 万门槛 | 港股通不支持，需本地券商 |
| 特色信号 | 北向资金、龙虎榜 | 南下资金、卖空比例 |
| 独有地雷 | ST/退市/面值退 | 老千股/仙股 |

**同一套选股规则直接套两个市场是错的**。详见 [6. A 股特色](6.%20A%20股特色.md) 和 [7. 港股特色](7.%20港股特色.md)。

## skill 输出形态：Top-down 四步

```mermaid
flowchart TD
    Q["用户自然语言查询<br/>『当前最值得关注的行业』<br/>『X 行业给我推 5 只』"]
    Q --> P["Socratic 画像<br/>stock-profile.yaml"]
    P --> I["宏观 + 行业筛选<br/>六维景气度"]
    I --> S["行业内 12 维打分"]
    S --> F["硬否决过滤<br/>A+HK 两套清单"]
    F --> OUT["三层候选池<br/>核心 5-15 · 观察 30-50 · 候选 100-200"]

    style OUT fill:#ffe8a8
```

这就是 skill 的骨架。本 wiki 的剩余页面把每一步拆开讲清楚。
