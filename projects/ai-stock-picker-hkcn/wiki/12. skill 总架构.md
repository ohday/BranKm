# skill 总架构

把前 11 页串起来，这一页给 skill 一张「**可实现的总装图**」：输入层（用户自然语言）→ 画像层（profile）→ 数据层（多源缓存）→ 打分层（12 维 + 硬否决）→ 输出层（三层池 + 五要素）。所有决策点、数据依赖、更新机制一次看全。读完这页你能开始造 skill。

## 两种典型查询的端到端追踪

### 查询 A：「当前最值得关注的行业是哪些」

```mermaid
sequenceDiagram
    participant U as 用户
    participant S as skill
    participant P as Profile
    participant D as 数据层
    participant E as 引擎

    U->>S: 『最值得关注的行业』
    S->>P: 读取 stock-profile.yaml
    P-->>S: market=A+HK, 周期=中线, 流派=quality+growth

    S->>E: 触发 Layer 1（美林时钟）
    E->>D: 获取 PMI / CPI / M2
    D-->>E: 宏观数据
    E-->>S: 当前处于『复苏后期』

    S->>E: 触发 Layer 2（六维景气度）
    E->>D: 行业财务 / 北向 / 估值 / 概念 / RPS / 一致预期
    D-->>E: 6 维度原始数据
    E->>E: 加权合成 + 滞涨过滤
    E-->>S: A 股 Top 5 + 港股 Top 5

    S->>U: 双市场行业榜<br/>（每个行业附说明与潜在个股）
```

### 查询 B：「消费行业给我推 5 只长线股」

```mermaid
sequenceDiagram
    participant U as 用户
    participant S as skill
    participant P as Profile
    participant D as 数据层
    participant E as 引擎

    U->>S: 『消费行业给我推 5 只长线股』
    S->>P: 读取 profile
    P-->>S: 修改 horizon=long_term → 激活长线权重

    S->>E: 锁定行业=食品饮料+商贸零售+家电<br/>（申万二级）
    E->>D: 获取行业内所有股票
    D-->>E: ~200 只股票

    E->>E: 硬否决过滤
    Note over E: ST / 退市 / 商誉>30% / 质押>80% 剔除
    E-->>E: 通过：~140 只

    E->>E: 12 维度打分（长线权重<br/>基本面 65%）
    E-->>E: 排序后 Top 10

    E->>E: 按 profile.exclude_industries 再过滤<br/>(去掉白酒)
    E-->>S: Top 5

    S->>U: 输出 5 只核心池候选<br/>含亮点 + 风险 + 催化剂
```

## Profile 的"活"机制

```mermaid
flowchart LR
    subgraph COLD["首次"]
        C1["Q1-Q15 Cold Start"] --> PV1["profile v1.0"]
    end

    subgraph MONTHLY["月度"]
        M1["5 题精简问卷"] --> PVN["profile v1.N"]
    end

    subgraph FEEDBACK["实时"]
        F1["接受推荐 → style.* 加权"]
        F2["拒绝 + 理由 → 维度权重微调"]
        F3["连续忽略行业 → 加入 exclude"]
        F4["主动搜索 → prefer_industries"]

        F1 & F2 & F3 & F4 --> PVN_LIVE["profile live"]
    end

    PV1 --> MONTHLY --> FEEDBACK
```

**关键设计**：
- **冷启动版本号 1.0**，每次月度问卷后增 0.1
- **行为校准不改版本号**，只改字段（低幅度 ±0.05）
- **重大事件触发 cold start 刷新**（用户资产变化、退休、市场危机）

## 输出规范

```mermaid
flowchart TB
    OUT_SHELL["输出骨架"]

    OUT_SHELL --> HEAD["头部：查询回应 + profile 摘要"]

    subgraph CORE["★ 核心池 5 只"]
        CORE_ITEM["每只：<br/>得分 · Top 3 维度 · 亮点 · 风险 · 催化"]
    end

    subgraph WATCH["○ 观察池 5-10 只"]
        WATCH_ITEM["简版：<br/>等待条件 + 已观察周数"]
    end

    subgraph CAND["△ 候选池 10-20 只"]
        CAND_ITEM["超简版：<br/>一句话描述"]
    end

    OUT_SHELL --> CORE --> WATCH --> CAND
    OUT_SHELL --> FOOTER["脚部：合规提示 + 交互引导"]
```

**新手模式** 默认只显示核心 5 + 观察池简版。**高级模式** 可展开所有细节。

**解耦设计**：
- **本 skill 只输出候选池**（不涉及买卖时点）
- **姐妹 skill 消费候选池 + 监控触发条件**
- 两者通过文件/DB 通信，**可独立使用**

## 关键参数汇总（skill 的"常量表"）

| 参数 | 默认值 | 依据 |
|------|-------|------|
| 候选池大小 | 100-200 | 散户可跟踪上限 |
| 观察池大小 | 30-50 | 深度研究上限 |
| 核心池大小 | 5-15 | 实盘聚焦 |
| Cold Start 题数 | 15 | < 10 分钟完成 |
| 月度精简题数 | 5 | 低负担 |
| 硬否决规则总数 | 19 (A 10 + HK 9) | |
| 商誉硬否决阈值 | 50% | 安信策略[^45] |
| 质押硬否决阈值 | 80% | 经济日报[^45] |
| 港股卖空硬否决 | > 10% | 风险保护 |
| M-Score A 股阈值 | -1.89 | 本土化[^45] |
| 行业滞涨过滤 | 90 日涨幅 > 20% | 华宝[^44] |
| 行为校准幅度 | ±0.05 | 避免震荡 |
| 黑名单持续 | 6 个月 | 避免重复纠结 |
| 核心池推荐数量（默认） | 5 | 新手上限 |

## 对新手的落地五步

```mermaid
flowchart TD
    STEP1["① 用 skill 跑出核心池 5 只<br/>（不急着买）"]
    STEP2["② 看懂每只的推荐理由<br/>（Top 3 维度贡献）"]
    STEP3["③ 检查硬否决是否漏判<br/>（偶尔数据延迟）"]
    STEP4["④ 持有 / 观察 1 个月<br/>再让 skill 重评"]
    STEP5["⑤ 反馈调整 profile<br/>（skill 会学习）"]

    STEP1 --> STEP2 --> STEP3 --> STEP4 --> STEP5 --> STEP1

    style STEP5 fill:#ffe8a8
```

**最容易的错误**：跳过 Layer 0 画像直接 Layer 3 打分 → 推荐完全不符合风险偏好。

## 最终小结

```mermaid
flowchart LR
    subgraph ESSENCE["skill 本质"]
        E1["不是『代我决策』"]
        E2["而是『帮我想清楚』"]
    end

    ESSENCE --> VALUE["给新手：<br/>结构化选股框架 +<br/>硬否决底线 +<br/>Socratic 自我认知"]

    VALUE --> OUTCOME["读者能与 skill 对话<br/>判断推荐是否靠谱<br/>不盲目跟从"]

    style OUTCOME fill:#ffe8a8
```

**skill 不承诺收益，承诺结构化**——这是它与"股神"的最大区别。读完这 12 页，你已经掌握 skill 的全部内部运作；下一步是去造一个（或者使用本项目产出的 skill）。

## 扩展方向（如果用户想自己造）

- **本 skill 不涉及**：组合构建（Markowitz/Risk Parity）、仓位管理、止损止盈策略
- **可以叠加的扩展**：
  - 回测框架（Qlib）
  - 实盘对接（富途 OpenAPI，港股）
  - 研报自动生成（TradingAgents-CN 框架）
  - 宏观预警（美林时钟自动监测）
