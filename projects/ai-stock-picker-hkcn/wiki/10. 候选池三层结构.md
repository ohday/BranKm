# 候选池三层结构

skill 的输出**不是一个简单的 Top-N 排行榜**。经过打分排序后，候选股票按"入场优先级"分成三层：核心池、观察池、候选池。这套三层结构来自专业机构的真实工作流，核心解决两个问题：**（1）新手如何避免"自选股变杂货铺"；（2）何时该升降级一只股票**。这一页把机构的四层体系简化为散户版三层，以及 skill 如何在输出中自然呈现。

## 三层池总架构

```mermaid
flowchart TB
    subgraph PIPE["筛选漏斗"]
        ALL["全市场 A+HK ~7500 只"]
        ALL --> CANDIDATE["候选池 100-200<br/>量化粗筛通过"]
        CANDIDATE --> WATCH["观察池 30-50<br/>经过深度研究"]
        WATCH --> CORE["★ 核心池 5-15<br/>实盘配置"]
    end

    CORE -.仓位.-> W1["50-60% 总仓位"]
    WATCH -.仓位.-> W2["0-10% 试仓"]
    CANDIDATE -.仓位.-> W3["0% 不持有"]

    style CORE fill:#ffe8a8
    style WATCH fill:#e3f2fd
    style CANDIDATE fill:#f5f5f5
```

**关键原则**：不是所有的候选池股票都值得买——候选池是"**值得跟踪**"，核心池才是"**值得买**"[^42]。

## 三层定位对比

| 层级 | 数量 | 门槛 | 跟踪深度 | 仓位分配 |
|------|:---:|------|---------|---------|
| **△ 候选池** | 100-200 | 量化粗筛 | 月度审查 | 0% |
| **○ 观察池** | 30-50 | 深度研究 | 周度审查 | 0-10%（试仓） |
| **★ 核心池** | 5-15 | 最高标准 | 日度跟踪 | 50-60% |

## 各层的筛选标准

### 候选池：量化粗筛

```mermaid
flowchart LR
    QUANT_IN["全市场"]

    QUANT_IN --> F1["财务硬门槛<br/>营收 > 0<br/>净利润 > 0 （长线）<br/>或 营收 CAGR > 15%"]
    F1 --> F2["行业合格<br/>来自六维景气 Top 10 行业"]
    F2 --> F3["技术面合格<br/>20MA 上穿 60MA 或价稳"]
    F3 --> F4["硬否决过滤<br/>A 股 10 条 + 港股 9 条"]
    F4 --> POOL_C["候选池"]
```

**候选池的本质**：所有**没有明显问题**的股票集合。门槛低，数量多。

### 观察池：深度研究

```mermaid
flowchart TB
    POOL_C["候选池 100-200"]

    POOL_C --> D1["行业地位<br/>与行业龙头对比"]
    POOL_C --> D2["3-5 年财报纵向分析<br/>营收/净利/ROE 连续性"]
    POOL_C --> D3["商业模式稳健性<br/>现金流质量"]
    POOL_C --> D4["技术面验证<br/>趋势持续 + 资金流向"]

    D1 & D2 & D3 & D4 --> RESEARCH["深度研究通过 ✓"]
    RESEARCH --> POOL_W["观察池 30-50"]
```

### 核心池：实盘配置标准

```mermaid
flowchart TD
    POOL_W["观察池"] --> CHECK{"核心池 5 条标准"}

    CHECK --> C1["行业龙头或细分冠军"]
    CHECK --> C2["连续 3 年盈利增长<br/>低负债 + 高现金流"]
    CHECK --> C3["朝阳产业<br/>新能源/AI/半导体/创新药"]
    CHECK --> C4["股价位于 60 日均线上方<br/>放量确认"]
    CHECK --> C5["有明确催化剂<br/>业绩预告 / 政策受益"]

    C1 & C2 & C3 & C4 & C5 --> POOL_CORE["★ 核心池"]

    style POOL_CORE fill:#ffe8a8
```

## 升降级规则（量化机制）

```mermaid
flowchart LR
    subgraph UP["升级"]
        U1["候选 → 观察<br/>通过深度研究"]
        U2["观察 → 核心<br/>催化剂兑现 + 技术确认"]
    end

    subgraph DOWN["降级 / 剔除"]
        D1["财务暴雷<br/>→ 直接剔除"]
        D2["行业景气反转<br/>→ 降级"]
        D3["技术趋势破位<br/>→ 降级"]
        D4["止损触发<br/>→ 剔除 + 加黑名单"]
    end
```

### 降级的明确触发条件

| 触发条件 | 动作 |
|---------|------|
| 季报营收/净利 QoQ < -10% | 核心池 → 观察池 |
| 单日跌幅 > -8% 且无明显利空 | 观察池 → 候选池 |
| 连续 2 个月北向净流出 > 10% | 核心池 → 观察池 |
| 命中硬否决任一条 | **直接剔除 + 加入黑名单 6 个月** |
| 季报被出具非标审计意见 | **直接剔除** |

**6 个月黑名单**：避免用户反复纠结已出问题的股票。6 个月后可重新评估。

## 新手友好的输出设计

skill 每次输出候选池时，**每只股票的呈现结构**必须包含五要素：

```mermaid
flowchart TB
    ITEM["一只推荐股票"]

    ITEM --> F1["① 为什么推它<br/>（一句话核心理由）"]
    ITEM --> F2["② 亮点<br/>（2-3 条数据支撑）"]
    ITEM --> F3["③ 风险<br/>（1-2 条反向提醒）"]
    ITEM --> F4["④ 催化剂<br/>（短期驱动事件）"]
    ITEM --> F5["⑤ 合规提示<br/>（仅供学习 非投资建议）"]
```

缺少 ③ 和 ⑤ 是新手 skill 最容易犯的错——**只讲好听的，不讲风险**，结果用户追高被套。

## 展示层的取舍

### 要不要输出目标价 / 止损位？

```mermaid
flowchart LR
    CHOICE["输出目标价 / 止损位？"]

    CHOICE --> PRO["Pro: 操作明确<br/>用户知道买卖点"]
    CHOICE --> CON["Con: 新手可能当成<br/>skill 的『投资建议』"]

    CON --> DEFAULT["默认关闭<br/>expert 用户可打开"]

    style DEFAULT fill:#ffe8a8
```

**默认关闭**。原因：
1. 新手易把 skill 输出当成"承诺"，追高被套后归罪于 skill
2. 选股 skill 不等于择时 skill，不该越位
3. 合规风险——给出具体价位可能触及"投资建议"监管

**高阶用户**可在 `profile.yaml` 里设 `show_target_price: true` 打开。

### 详细度三档

| 档位 | 对象 | 显示内容 |
|------|------|---------|
| `newbie` | 新手（默认） | 得分 + Top 3 维度 + 亮点/风险/催化（一段话） |
| `intermediate` | 中级 | + 全部 12 维度得分 + 同行业横向比较 |
| `expert` | 专业 | + 因子 IR / 行业中性化后分数 / 历史回测 |

## 机构 vs 散户的取舍

```mermaid
flowchart LR
    subgraph INST["机构（量化团队）"]
        I1["4 级：基础库 + 备选库 + 核心库 + 禁买库"]
        I2["因子打分 + 风险预算 + 投研会议"]
        I3["每日再平衡"]
        I4["覆盖 200-500 只股票"]
    end

    subgraph RETAIL["散户（本 skill 目标）"]
        R1["3 级：候选 + 观察 + 核心"]
        R2["每月重评 + 用户确认"]
        R3["每周跟踪"]
        R4["聚焦 10-30 只"]
    end

    INST -.简化.-> RETAIL
```

**简化的关键**：砍掉"禁买库"（合并到黑名单），机构 4 级变散户 3 级。保留核心的升降级逻辑[^42]。

## 散户常见错误

```mermaid
flowchart TD
    BAD1["❌ 杂货铺现象<br/>自选股一堆<br/>从不整理"]
    BAD2["❌ 核心池过大<br/>10 只以上无法<br/>深度跟踪"]
    BAD3["❌ 没有降级机制<br/>只进不出"]
    BAD4["❌ 混淆配置和交易<br/>核心池 ≠ 今天就买"]

    BAD1 --> FIX1["skill 限制：<br/>不进三层池的<br/>不推荐跟踪"]
    BAD2 --> FIX2["skill 默认核心池 ≤ 5"]
    BAD3 --> FIX3["skill 每月自动审查<br/>降级条件"]
    BAD4 --> FIX4["skill 明确区分<br/>选股 vs 择时"]

    style BAD1 fill:#ffcccc
    style BAD2 fill:#ffcccc
    style BAD3 fill:#ffcccc
    style BAD4 fill:#ffcccc
```

## 与 skill 对话的约束

```mermaid
flowchart LR
    USER_ACT["用户行为"]

    USER_ACT --> A1["『我不喜欢白酒』"]
    A1 --> R1["skill 下次自动排除白酒行业"]

    USER_ACT --> A2["『加入核心池』反复点击某类"]
    A2 --> R2["反向推断流派倾向"]

    USER_ACT --> A3["『为什么推这只』"]
    A3 --> R3["展开 Top 3 维度细节<br/>+ 历史相似案例"]

    USER_ACT --> A4["『换一个』"]
    A4 --> R4["从观察池提拔下一只"]
```

这就是 "**Socratic 引导 + 候选池**" 交互的完整闭环——画像驱动推荐，反馈更新画像，形成良性循环。

## 关键结论

1. **三层池 ≠ 排行榜**——优先级分层是为了避免"自选股杂货铺"
2. **降级规则必须量化**——否则只进不出
3. **每只推荐含五要素**——缺一不可（核心理由 · 亮点 · 风险 · 催化剂 · 合规）
4. **目标价默认关闭**——合规与"越位择时"双重考虑
5. **核心池 ≤ 5**（新手）—— 跟不过来就等于没跟
