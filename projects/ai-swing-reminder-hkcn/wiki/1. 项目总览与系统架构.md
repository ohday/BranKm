# 项目总览与系统架构

本 wiki 回答一个具体问题：**如何在 OpenClaw 上跑一个 AI 驱动的波段交易助手，覆盖 A 股与港股，A 股只发提醒、港股可选下单**。面向中级量化用户（懂回测、懂 Python、不愿从零造轮子）。

## 核心架构决策（3 条）

**决策 1：不做 A 股自动下单**[^32]

A 股 2024-10-08 起的《程序化交易管理规定》对散户同等适用，未报备即下单有风险。纯提醒模式合规，也天然避免 LLM 幻觉带来的资金损失。港股的 Futu / Longbridge 是持牌券商 API，通过 Telegram 人工二次确认后下单合法合规。

**决策 2：主推路线 A（LLM 规则生成），辅以路线 C（Multi-Agent）**[^28]

```mermaid
flowchart LR
    NL["自然语言偏好<br/>「20日均线 + 量能放大」"] --> Claude["Claude<br/>生成代码"]
    Claude --> Code["Backtrader<br/>策略类"]
    Code --> BT["回测 5 年"]
    BT --> R["夏普 1.35<br/>回撤 14.8%"]
    R --> Claude2["Claude<br/>解读 + 调参建议"]
    Claude2 --> Deploy["OpenClaw Skill<br/>上线"]
    Deploy -->|触发| Notify["Telegram + 企业微信"]

    subgraph Aux ["路线 C · 周末辅助"]
        TA["TradingAgents-CN<br/>多 agent 辩论"]
    end

    Deploy -.周末 Watchlist 复核.-> TA
    TA -.二次意见.-> Notify
```

路线 B（强化学习）和 D（LLM 因子挖掘）在波段 + 中级用户场景下**复杂度不匹配**——过拟合风险高且学习曲线陡峭；详见 [4. AI 构建策略的四条路线](4.%20AI%20构建策略的四条路线.md)。

**决策 3：腾讯云香港轻量单节点起步**[^31]

不做双节点拆分（大陆 + 香港），单香港节点就能同时访问 Tushare Token（HTTPS）、AkShare 爬东财（有一定延迟但日频够用）、Futu OpenAPI（延迟 1-3ms）、Telegram（海外直连）、DeepSeek/Claude API（都可达）。月开销 ≈ $12。

对比 <¥500/月 的约束还余 60%+ 预算——这部分可以升级 Tushare 至 5000 积分解锁分钟线，或开通 Longbridge L2 行情。详细成本结构见 [10. 端到端 Pipeline 模板](10.%20端到端%20Pipeline%20模板.md)。

## 关键路径时序

从用户第一次跑到线上系统每天按时推送的完整时序：

```mermaid
sequenceDiagram
    actor U as 用户
    participant Cl as Claude
    participant Q as Qlib/Backtrader
    participant OC as OpenClaw
    participant Src as 数据源
    participant IM as Telegram/企业微信
    participant Futu as Futu OpenAPI

    Note over U,IM: ——— 一次性：策略建设（1~2 周）———
    U->>Cl: 自然语言描述偏好
    Cl-->>U: 生成策略代码
    U->>Q: 5 年回测 + WF 验证
    Q-->>Cl: 回测报告
    Cl-->>U: 解读 + 调参建议
    U->>OC: 封装为 Skill

    Note over U,IM: ——— 每日：自动运行 ———
    OC->>Src: 15:10 cron 触发
    Src-->>OC: 日 K + 因子数据
    OC->>OC: 本地计算信号 + 去重 + 分级
    alt L1 信号
        OC->>IM: 企业微信推送
    else L2/L3
        OC->>IM: 企业微信 + Telegram + Bark
    end

    Note over U,Futu: ——— 港股可选：人工确认下单 ———
    IM->>U: 收到提醒
    U->>IM: 回复 /tc_BUY_0700_100_315
    IM->>OC: 解析指令
    OC->>IM: 二次确认 YES?
    U->>IM: YES
    IM->>OC: 确认
    OC->>Futu: place_order
    Futu-->>OC: order_id
    OC->>IM: "✅ 订单已提交"
```

## 一句话总结

> **用 LLM 帮自己写策略代码 + 人工 review + 历史回测 + walk-forward 验证 + OpenClaw 上线 + 提醒到 Telegram。A 股至此结束；港股在此基础上追加 Futu API 人工确认下单层。**
