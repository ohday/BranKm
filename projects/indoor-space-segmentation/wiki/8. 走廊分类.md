# 走廊分类

分割产生区域标签后，需要为每个区域指定**类型**：房间、走廊、大厅、交叉口等。走廊是建筑中最具辨识度的类型——狭长、窄小、连接多个空间。本页介绍四种检测方法，以及如何处理分支走廊和 T 形交叉口<sup>[[10]](#sources)</sup>。

## 四种检测方法

```mermaid
flowchart TD
    subgraph "方法一：长宽比"
        AR["最小包围盒\n长宽比 > 3:1\n→ 走廊"]
    end
    subgraph "方法二：多尺度腐蚀"
        MSE["小幅腐蚀：走廊保留\n大幅腐蚀：走廊消失\n差集 = 走廊区域"]
    end
    subgraph "方法三：距离变换 + 紧凑度"
        DTC["距离变换最大值低\n× 紧凑度低\n→ 走廊"]
    end
    subgraph "方法四：图度数"
        GD["区域连接\n≥ 3 个其他区域\n→ 走廊"]
    end

    AR & MSE & DTC & GD --> CLASSIFY["区域类型：\n房间 | 走廊 | 大厅 | 交叉口"]
```

## 分类指标

以下五个几何特征可以可靠地区分房间和走廊<sup>[[10]](#sources)</sup>：

| 特征 | 房间（典型值） | 走廊（典型值） | 公式 |
|------|---------------|---------------|------|
| **长宽比** | < 2:1 | **> 3:1** | OBB 长边 / OBB 短边 |
| **紧凑度** | > 0.05 | **< 0.02** | 面积 / 周长² |
| **距离变换最大值** | 大（> 2m） | **小（< 1.5m）** | 区域内 max(Ō_min) |
| **图度数** | 1–2 | **≥ 3** | 相邻区域数量 |
| **截面方差** | 高 | **低** | var(沿骨架的 Ō_min) |

**建议**：使用 **3 个以上特征投票** 进行稳健分类。任何单一特征都存在失败情况。

## 方法一：长宽比分析（最可靠）

```mermaid
flowchart TD
    ZONE["分割后的区域\n（体素集合）"]
    ZONE --> OBB["计算有向\n包围盒（OBB）"]
    OBB --> RATIO["长宽比 =\n长边 / 短边"]
    RATIO --> CLASS{"比值 > 3？"}
    CLASS -->|"是"| CORR["走廊候选"]
    CLASS -->|"否"| ROOM["房间或大厅"]
    
    CORR --> VERIFY{"DT_max < 1.5m？"}
    VERIFY -->|"是"| CORRIDOR["✅ 走廊"]
    VERIFY -->|"否"| HALL["大厅（宽阔的狭长空间）"]
```

二次距离变换检查可以防止宽阔的狭长空间（如会议室、展厅）被误分类为走廊。

## 方法二：多尺度腐蚀差分

```mermaid
flowchart LR
    ORIG["原始自由空间"]
    ORIG --> E1["腐蚀半径\nr = 0.3m\n（门宽级别）"]
    ORIG --> E2["腐蚀半径\nr = 1.0m\n（房间宽度级别）"]
    E1 --> DIFF["差集：\n小腐蚀后保留\n但大腐蚀后消失"]
    E2 --> DIFF
    DIFF --> CORR["= 走廊宽度\n区域"]
```

走廊在小幅腐蚀（比门宽）后会保留，但在大幅腐蚀（比房间窄）后会消失。这两次腐蚀结果的差集可以隔离出走廊宽度的空间。

## 方法三：基于骨架的分析

自由空间的[三维骨架](1. 射线距离到标量场.md)天然编码了建筑拓扑结构<sup>[[14]](#sources)</sup>：

```mermaid
flowchart TD
    FREE["自由空间\n体素网格"]
    FREE --> THIN["三维细化\n（保持峡部）"]
    THIN --> SKEL["单体素骨架"]
    SKEL --> ANALYZE["分析骨架\n线段"]
    ANALYZE --> LONG["长线性段\n→ 走廊"]
    ANALYZE --> STAR["星形子图\n→ 房间"]
    ANALYZE --> BRANCH["分支点\n→ 交叉口"]
    ANALYZE --> LEAF["叶端点\n→ 死胡同房间"]
```

**骨架分类规则**：
- **线性段**（低曲率，长度 > 3× 宽度）→ 走廊
- **星形子图**（多条短分支汇聚于中心）→ 房间
- **分支点**（骨架图中度数 ≥ 3）→ 交叉口区域
- **叶节点**（度数 1）→ 死胡同房间或储藏间

## T 形交叉口与分支走廊

```mermaid
flowchart LR
    subgraph "T 形交叉口"
        C1["走廊 A"] --- J1["交叉口\n区域"]
        J1 --- C2["走廊 B"]
        J1 --- C3["走廊 C"]
    end
    subgraph "L 形弯道"
        C4["走廊 D"] --- B1["弯道\n（同一区域）"]
        B1 --- C5["走廊 E\n（可能与 D 合并）"]
    end
```

**处理策略**：
1. 检测骨架分支点 → 标记为**交叉口区域**
2. 在每个分支点处将走廊拆分为独立的线性段
3. 每个线性段 = 独立的走廊子区域
4. 分支点区域 = 交叉口区域（通常为小型方形/圆形区域）
5. L 形弯道（双向分支且角度 < 120°）可保持为单一走廊

## 方法四：各向异性场

[各向异性场 A](1. 射线距离到标量场.md) 提供了最直接的走廊信号：

```
A(v) = max(ray_distances) / min(ray_distances)
```

- 走廊体素：A ≫ 3（沿一个方向长，垂直方向窄）
- 房间体素：A ≈ 1–2（各方向大致相等）
- 靠墙体素：A ≫ 1 但 Ō_min 很低（需过滤掉）

**流程**：阈值 A > 3 且 Ō_min > 0.3m → 走廊候选体素。聚类为连通区域 → 走廊区域。

## 综合分类流程

```mermaid
flowchart TD
    ZONE["分割产生的\n区域"]
    ZONE --> F1["计算：\n长宽比、紧凑度、\nDT_max、图度数、\n平均各向异性"]
    F1 --> VOTE["5 个特征\n多数投票"]
    VOTE --> C{"分类结果"}
    C -->|"4-5 票走廊"| CORR["走廊"]
    C -->|"4-5 票房间"| ROOM["房间"]
    C -->|"混合 / 大面积 + 狭长"| HALL["大厅"]
    C -->|"高度数 + 紧凑"| JUNCTION["交叉口"]
```
